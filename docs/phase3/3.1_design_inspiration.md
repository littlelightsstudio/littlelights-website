# 3.1 Design-Inspiration & Stilrichtung
**Little Lights Studio Website | Stand: April 2026**

---

## Design-Vision

**Hell, freundlich, stilvoll, mit Substanz.** Kein dunkler Kino-Look (davon kommen wir ab), kein generischer WordPress-Template-Look. Stattdessen: eine Website die sich anfühlt wie ein gut gemachter Film. Ruhig, selbstbewusst, mit punktuellen Momenten die überraschen.

### Design-Prinzipien

1. **Substanz vor Effekt.** Jeder visuelle Entscheid hat einen Grund. Keine Animation um der Animation willen.
2. **Freundlich und professionell.** Heller Grundton, viel Weißraum, aber warm (nicht steril).
3. **Filmisch ohne dunkel zu sein.** Cinematische Bildsprache (Stills, Loops) auf hellem Grund. Der Kontrast zwischen hellem Layout und filmischem Content erzeugt Spannung.
4. **Punktuelle Wow-Momente.** 2-3 smart integrierte Effekte die den User beeindrucken und gut führen, nicht 50 Animationen die ablenken.
5. **Charmant.** Wie die Tonalität der Texte: ein Augenzwinkern an den richtigen Stellen.

## Konkrete Effekte und Interaktionen

### Empfohlen (smart, stilvoll, funktional)

**1. Video-Reveal beim Scrollen**
Case Study Bilder sind zunächst still, werden beim Scrollen in den Viewport zu kurzen Video-Loops. Subtil, filmisch, zeigt die Arbeit ohne dass der User klicken muss.

**2. Parallax-Textebenen**
Texte und Bilder bewegen sich mit leicht unterschiedlicher Geschwindigkeit beim Scrollen. Erzeugt Tiefe. Dezent eingesetzt (nur auf Homepage und Agency-Landingpage).

**3. Smooth Scroll mit Fade-In**
Elemente erscheinen sanft beim Scrollen (Opacity 0→1, leichter Y-Translate). Kein hartes Einblenden. Gibt der Seite einen cinematischen Rhythmus.

**4. Cursor-Interaktion auf Projekt-Thumbnails**
Beim Hover über ein Projekt erscheint ein Custom-Cursor ("Play" oder "Ansehen"). Subtiler Zoom-Effekt auf dem Bild. Zeigt: hier gibt es mehr zu entdecken.

**5. Zahlen-Counter**
Die Kennzahlen (12+ Jahre, 200+ Projekte) zählen hoch wenn sie in den Viewport scrollen. Einmaliger Effekt, erzeugt einen Moment der Aufmerksamkeit.

**6. Partnerschafts-Timeline**
Eine horizontale Timeline die beim Scrollen aufgebaut wird: Strabag 2014 → ... → Karrierestories 2023 → 150+ Episoden. Storytelling durch Scroll-Interaktion.

### Vermeiden (Effekthascherei)

- Parallax auf jeder Sektion (wird schnell ermüdend)
- Cursor-Trail-Effekte (ablenkend, performancelastig)
- Text-Morphing/Glitch-Effekte (passen nicht zur Tonalität)
- Infinite Scroll ohne Zweck (User verliert Orientierung)
- Ladeanimationen die länger dauern als das Laden selbst
- Horizontales Scrollen als Hauptnavigation (frustrierend auf Mobile)

## Farbkonzept (Update)

Abweichung vom dunklen Look. Neues Konzept:

### Primär: Hell und freundlich
```
Hintergrund:        #FAFAFA oder #F5F5F5 (warmes Weiß, nicht steril)
Text Primary:       #1B1E29 (LL Grey, für Headlines und Body)
Text Secondary:     rgba(27, 30, 41, 0.6) (für Subtexte, Labels)
```

### Akzent: LL Blue
```
Primary Blue:       #22528e (CTAs, Links, aktive Elemente)
Blue Hover:         #1a4170 (dunklere Variante für Hover)
Blue Light:         rgba(34, 82, 142, 0.08) (Hintergründe, Tags)
```

### Kontrast-Sektionen: Dunkle Bereiche für filmische Momente
```
Dark Section BG:    #1B1E29 (LL Grey)
Dark Section Text:  #FAFAFA
```

**Strategie:** Die Seite ist überwiegend hell. Aber einzelne Sektionen (z.B. der Video-Hero, die Partnerschafts-Sektion, der Footer) nutzen den dunklen Hintergrund. Das erzeugt Rhythmus und lässt die filmischen Inhalte (Video-Stills, Loops) besonders gut wirken.

## Typografie

```
Headline Font:      Satoshi Bold/Black (700/900)
Body Font:          Satoshi Regular/Medium (400/500)
Small/Labels:       Satoshi Medium (500), Uppercase, Letter-Spacing 0.1-0.2em
Accent (optional):  Ein Serif-Font für Claims/Quotes (z.B. die englischen Claims)
                    Kandidaten: Crimson Pro, Source Serif, oder bei Satoshi bleiben
```

**Typografische Hierarchie:**
- H1: Satoshi Black, 48-72px (Desktop), fluid mit clamp()
- H2: Satoshi Bold, 32-48px
- H3: Satoshi Bold, 24-32px
- Body: Satoshi Regular, 16-18px, Zeilenhöhe 1.6-1.7
- Small/Labels: Satoshi Medium, 11-13px, Uppercase, Tracking

## Technische Umsetzung der Effekte

| Effekt | Technik | Performance |
|--------|---------|-------------|
| Smooth Scroll | CSS `scroll-behavior: smooth` + Lenis Library | Leichtgewichtig |
| Fade-In on Scroll | Intersection Observer API (nativ) | Kein Library nötig |
| Video-Reveal | Intersection Observer + Lazy-Load Vimeo/Bunny | Performant |
| Parallax (dezent) | CSS `transform: translateY()` mit Scroll-Listener | Leicht, GPU-beschleunigt |
| Zahlen-Counter | Intersection Observer + requestAnimationFrame | Minimal |
| Cursor-Effekt | CSS + leichter JS-Listener | Nur Desktop |
| Timeline | GSAP ScrollTrigger (einziges externes Library für Animation) | Bewährt, performant |

**Gesamt-Ansatz:** So wenig externe Libraries wie möglich. Native Browser-APIs (Intersection Observer, CSS Transitions) für 90% der Effekte. GSAP ScrollTrigger nur für die Timeline-Animation. Kein Three.js, kein Framer Motion, kein Overhead.
