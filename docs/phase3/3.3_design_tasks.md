**Little Lights Studio | Stand: 14. April 2026**

---

## Grundprinzipien

- **Templates, nicht Einzelstücke.** Jede Sektion wird als abstrahierte, wiederverwendbare Komponente gedacht — einsetzbar auf Homepage und allen weiteren Seiten.
- **CMS-ready.** Datenstrukturen von Anfang an mitdenken. Jede Komponente soll in Payload CMS abbildbar sein.
- **Code auf Englisch.** Variablen, Klassen, Kommentare — alles English. Kommunikation bleibt Deutsch.
- **Qualität vor Geschwindigkeit.** Nichts schnell im Pfusch. Jede Komponente so durchdacht, dass sie skaliert.
- **Isoliert bauen.** Jede Komponente erst als eigene HTML-Datei, testen, dann integrieren.

### Horizontaler Copper-Divider (wiederverwendbar)

Subtile horizontale Linie mit Copper-Gradient (transparent → Copper → transparent). Einsetzen wo gleichfarbige Sektionen aufeinandertreffen - erzeugt elegante Trennung ohne den Fluss zu brechen. Keine diagonalen Section-Divider (Ausnahme: Footer).

### Bereichs-Farbschemen

Drei Bereiche, drei dezent unterschiedliche Farbwelten — so erkennt der Besucher sofort, wo er sich befindet:

| Bereich | Akzentfarbe | Einsatz |
|---------|-------------|---------|
| Agency | Copper #C8956C (Primär) | CTAs, Divider, Labels |
| Reels & Stories | TBD — wärmer/energetischer? | Eigene Akzente |
| Creative Studio | TBD — kühler/poetischer? | Eigene Akzente |

Basis (Navy + Cream) bleibt gleich. Nur der Akzent wechselt dezent.

## 4. Diagonal Slider

**Status:** Gelockt für 3-Panel-Version. Muss abstrahiert werden.

### Was steht
- Drei Panels mit clip-path Diagonalen
- Copper Divider, Hover-Expansion
- Claims pro Panel: Agency / R&S / Creative

### Open Tasks
- [ ] **Abstraktion:** Slider muss mit 3–6 Panels funktionieren
- [ ] **Neue Logik für 4+ Panels:**
  - Erstes Panel: volle Breite sichtbar (aktiv)
  - Restliche Panels: auf ~15% komprimiert
  - Komprimierte Panels zeigen nur Copper-Titel, schräg am Divider
  - On Hover (Desktop) / On Click (Mobile): Panel expandiert, aktives Panel komprimiert
- [ ] **Mobile:** Nur ein Panel sichtbar, Rest als komprimierte Streifen mit schrägen Titeln
- [ ] **Wiederverwendung:** Slider soll auch für Erklärungen, Aufzählungen, Service-Übersichten nutzbar sein

### CMS-Datenstruktur
```
DiagonalSlider:
  - panels: Array of {
      label: string (Copper-Label)
      headline: localized string
      body: localized string
      image: image reference
      ctaLabel: localized string
      ctaUrl: string
      accentColor?: string (optional override)
    }
  - defaultActiveIndex: number (default: 0)
```

## 5. Ausgewählte Arbeiten

**Status:** Neu konzipiert — ersetzt das bisherige Case-Study-Layout

### Konzept Homepage (5 Projekte, Pyramiden-Hierarchie)

```
┌─────────────────────────────────────┐
│         1. HERO PROJECT             │  ← Größtes Projekt, volle Breite
│         (16:9 oder 2:1)             │     Cineastisch, dominant
└─────────────────────────────────────┘
┌────────────────┐ ┌────────────────┐
│   2. PROJEKT   │ │   3. PROJEKT   │  ← Halbe Breite, gleiche Höhe
│                │ │                │
└────────────────┘ └────────────────┘
┌──────────┐ ┌──────────┐ ┌──────────┐
│ 4. PROJ  │ │ 5. PROJ  │ │ 6. PROJ  │  ← Drittel-Breite
└──────────┘ └──────────┘ └──────────┘
```

### Design-Regeln
- Bilder im cineastischen Format (16:9 oder 2:1) — nicht quadratisch
- Jedes Projekt zeigt: Titel, Kunde, Kategorie-Tag, Filmstill/Loop
- Hover/Click-Verhalten: TBD (später definieren)
- Diagonaler Copper-Accent irgendwo (konsistent mit Designsprache)

### Future Feature: Category Toggle
- Buttons/Tabs über dem Grid: Case Studies | Imagefilme | Employer Branding | Werbespots | Reels & Stories
- Toggle wechselt die 5 angezeigten Projekte (Animation: Crossfade oder Shuffle)
- Dezent unterschiedliche Akzentfarben pro Kategorie:
  - Case Studies / Premium: Copper (Standard)
  - Agency: TBD
  - Reels & Stories: TBD

### CMS-Datenstruktur
```
FeaturedProjects (Homepage):
  - projects: Array of {
      reference: relation to CaseStudy or RSProject
      size: 'hero' | 'medium' | 'small'
      position: number (1-6)
    }
  - categories: Array of {
      label: localized string
      filter: string (maps to category field)
      accentColor?: string
    }

ProjectCard (wiederverwendbar):
  - title: string
  - client: string
  - category: string
  - thumbnail: image reference
  - videoLoop?: video reference (optional, for hover)
  - url: string
```

### Auf weiteren Seiten
- Gleiches Karten-System, aber als Grid (2er, 3er, oder 4er Raster)
- Filter nach Kategorie
- Pagination oder "Mehr laden"

## 11. Mobile Responsive

**Status:** Ausstehend — wird nach Desktop-Design pro Komponente angegangen

### Grundsätze
- Jede Komponente bekommt ein eigenes Mobile-Konzept
- Touch-optimiert (kein Hover-abhängiges Verhalten ohne Fallback)
- Sticky Mobile CTA-Button nach Scroll über Hero
- Performance: Videos → Stills, weniger Animationen

### Pro Komponente (Michaels Ideen kommen hier rein)
- Navigation: Hamburger + Fullscreen-Overlay?
- Hero: Standbild statt Video, Diagonale anpassen
- Slider: Ein Panel aktiv, Rest als komprimierte Streifen
- Projekte: Vertikal gestapelt oder horizontal scrollbar
- Logos: Horizontal scrollbar
- Stats: 2x2 Grid
- Testimonials: Swipe-Slider
- Footer: Gestapelt, Logo oben
