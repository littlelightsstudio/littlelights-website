## Warum diese Kombination

### Payload CMS + Next.js

Payload 3.x ist nativ in Next.js integriert. Das bedeutet:
- **Ein Projekt, ein Deployment.** CMS-Admin und Website laufen in derselben Anwendung. Kein separater API-Server, keine Synchronisation zwischen zwei Systemen.
- **Server-Side Rendering (SSR) und Static Generation (SSG)** je nach Seite. Die Homepage kann statisch generiert werden (schnell), Case Studies können on-demand gerendert werden (immer aktuell).
- **React-basiert.** Flexibel für interaktive Elemente (Video-Player, Filter, Slider) ohne zusätzliche Frameworks.
- **App Router (Next.js 15)** für modernes Routing, Layouts und Streaming.

### Tailwind CSS

- Utility-first, kein Custom-CSS-Chaos
- Perfekt für responsive Design (mobile-first Breakpoints eingebaut)
- Dark Background als Default leicht umsetzbar
- Satoshi Font über `@font-face` einbinden, in Tailwind-Config registrieren

### PostgreSQL

- Stabiler als MongoDB für strukturierte Daten (Case Studies mit definierten Feldern)
- Payload unterstützt beides, aber PostgreSQL ist für diesen Anwendungsfall besser geeignet
- Relationale Queries (z.B. "alle Projekte in Kategorie EB") sind effizienter

### Hetzner Cloud

- **Standort:** Falkenstein oder Nürnberg, Deutschland
- **Nachhaltigkeit:** Erneuerbare Energie, eigene Rechenzentren
- **Kosten:** CX22 (2 vCPU, 4GB RAM, 40GB SSD) = ca. 5,49€/Monat. Reicht für die Website locker.
- **Europäisch:** DSGVO-konform, Daten in Deutschland
- **Performance:** Schnelle Anbindung, gute Latenz für DACH-Besucher

### Docker + Coolify

- **Coolify** ist ein Self-Hosted Deployment-Tool (wie Vercel, aber auf dem eigenen Server)
- Push to GitHub → automatisches Deployment
- SSL-Zertifikate automatisch (Let's Encrypt)
- Kein manuelles SSH-Deployment nötig
- Coolify selbst ist Open Source und kostenlos

### Medien-Handling

**Bilder:**
- Payload hat eingebaute Bildverarbeitung (Resize, Crop, WebP-Konvertierung)
- Speicherung auf Hetzner Object Storage (S3-kompatibel, ca. 1€/Monat für die Datenmenge einer Firmenwebsite)
- Alternative: Cloudflare R2 (großzügiger Free-Tier, aber US-basiert)
- Fallback: Lokal auf dem VPS (einfachste Lösung, aber weniger skalierbar)

**Videos:**
- Kein Self-Hosting. Videos bleiben auf Vimeo oder YouTube.
- Embed per URL im CMS (Payload Video-Embed-Feld)
- Lazy Loading: Vimeo/YouTube iframes werden erst geladen wenn der User scrollt (Performance)
- Placeholder: Thumbnail-Bild bis zum Laden des Players

### Analytics

- **Kein Google Analytics.** Datenschutzfreundlich, kein Cookie-Banner nötig.
- **Plausible** (gehostet, 9€/Monat) oder **Umami** (Self-Hosted auf Hetzner, kostenlos)
- Beide DSGVO-konform, leichtgewichtig, kein Performance-Impact

### E-Mail für Kontaktformular

- **Resend** (kostenloser Tier: 100 E-Mails/Tag, reicht locker) oder **Postmark**
- Zuverlässiger als eigener SMTP-Server
- Einfache Integration in Next.js Server Actions

## Content-Types in Payload

### Case Study (Agency)

```typescript
{
  slug: 'case-studies',
  fields: [
    { name: 'title', type: 'text', required: true },
    { name: 'client', type: 'text', required: true },
    { name: 'categories', type: 'select', hasMany: true,
      options: ['Employer Branding', 'Imagefilm', 'Werbespot', 
                'Branded Entertainment', 'Nachhaltigkeit'] },
    { name: 'shortDescription', type: 'textarea', required: true },
    { name: 'challenge', type: 'richText' },
    { name: 'approach', type: 'richText' },
    { name: 'result', type: 'richText' },
    { name: 'mainVideo', type: 'text' },         // Vimeo/YouTube URL
    { name: 'additionalVideos', type: 'array',
      fields: [{ name: 'url', type: 'text' }] },
    { name: 'gallery', type: 'upload', hasMany: true },
    { name: 'credits', type: 'richText' },
    { name: 'period', type: 'text' },
    { name: 'testimonial', type: 'group',
      fields: [
        { name: 'quote', type: 'textarea' },
        { name: 'author', type: 'text' },
        { name: 'role', type: 'text' }
      ]
    },
    { name: 'isFeatured', type: 'checkbox' },     // Full Case Study vs. Minimal
  ],
  admin: {
    defaultColumns: ['title', 'client', 'categories', 'isFeatured']
  }
}
```

Für Texte die der Auftraggeber selbst ändern können muss (Hero-Texte, Claims, Zahlen):

```typescript

  type: 'global',
  fields: [
    { name: 'heroH1', type: 'text' },
    { name: 'heroSubline', type: 'text' },
    { name: 'brandStatement', type: 'text' },
    { name: 'statsYears', type: 'number' },
    { name: 'statsProjects', type: 'number' },
    { name: 'statsClients', type: 'number' },
    { name: 'statsTeam', type: 'number' },
    // ... weitere editierbare Felder
  ]
}
```

### Zweisprachigkeit

Payload hat `localization` eingebaut:

```typescript
// payload.config.ts
localization: {
  locales: [
    { label: 'Deutsch', code: 'de' },
    { label: 'English', code: 'en' }
  ],
  defaultLocale: 'de',
}
```

Jedes Textfeld wird automatisch pro Sprache pflegbar. Im Admin sieht Michael einen Sprach-Switcher und kann DE und EN separat einpflegen.

## Deployment-Workflow

```
1. Code ändern (lokal oder Claude Code)
2. Git push to GitHub
3. Coolify erkennt Push automatisch
4. Docker-Image wird gebaut
5. Neue Version wird deployed
6. Zero-Downtime (Blue-Green Deployment)
```

Für Content-Änderungen (neue Case Study, Text-Update):
```
1. In Payload Admin einloggen (/admin)
2. Content bearbeiten
3. Speichern
4. Sofort live (kein Deployment nötig)
```
